# Issues Tracking

This folder contains tracked issues, bugs, and improvements for the multi-agent system.

## Structure

- **`active-issues.md`** — Currently active issues being tracked
- **`resolved/`** — Archive of resolved issues (moved here when closed)
- **`templates/`** — Issue templates for consistent reporting

## Issue Lifecycle

1. **New Issue**: Added to `active-issues.md`
2. **In Progress**: Issue status updated to "In Progress"
3. **Resolved**: Issue moved to `resolved/` folder with resolution date
4. **Closed**: Issue archived with final status

## Adding Issues

When adding a new issue to `active-issues.md`:
- Use sequential numbering (Issue #N)
- Include: Severity, Impact, Status, Problem, Solution, References
- Link to related tasks, documentation, or code
