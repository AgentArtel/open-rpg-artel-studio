# File-to-Agent Ownership Map

Every file in the repo maps to exactly one agent. When in doubt, check this file.

- `AGENTS.md` — universal source of truth

## Cursor (Implementation Specialist)

### Agent System
- `src/agents/core/**` — AgentRunner, LLM client abstraction, lane queue
- `src/agents/skills/**` — game command tool definitions (move, look, say, wait, emote)
- `src/agents/perception/**` — PerceptionEngine (game state → text)
- `src/agents/memory/**` — AgentMemory (conversation buffer, persistence)
- `src/agents/bridge/**` — GameChannelAdapter, RPGJS ↔ agent wiring

### Game Module (RPGJS autoload structure)
- `main/events/**` — NPC/Event classes (@EventData)
- `main/maps/**` — map classes (@MapData)
- `main/database/**` — items, weapons, skills (@Item, @Weapon, etc.)
- `main/spritesheets/**` — spritesheet definitions + PNG images
- `main/gui/**` — GUI components (Vue/React)
- `main/sounds/**` — audio assets
- `main/worlds/**` — Tiled .tmx/.tsx map files, tileset images
- `main/player.ts` — player lifecycle hooks
- `main/server.ts` — server hooks
- `main/client.ts` — client hooks
- `main/sprite.ts` — client sprite hooks
- `main/scene-map.ts` — scene/camera hooks

### Configuration
- `src/config/**` — agent personality YAML files
- `.cursor/rules/**` — Cursor-specific rules files
